## Overview

ultraenv can generate TypeScript type definitions from your environment schema, enabling:

- **Full autocomplete** for `process.env.YOUR_VAR`
- **Type checking** — catch missing or wrong-typed env vars at compile time
- **Documentation** — JSDoc comments from schema descriptions
- **Tool integration** — JSON Schema for editors and validators

---

## Quick Start

```bash
# Generate declaration file
ultraenv typegen --format declaration --out src/env.d.ts

# Generate TypeScript module
ultraenv typegen --format module --out src/env.ts

# Generate JSON Schema
ultraenv typegen --format json-schema --out env.schema.json
```

## Declaration Files (.d.ts)

Declaration files augment the `NodeJS.ProcessEnv` interface, giving you autocomplete for `process.env`:

### Generate

```bash
ultraenv typegen --format declaration --out src/env.d.ts
```

### Output

```typescript
// Auto-generated by ultraenv — DO NOT EDIT
// Generated: 2024-01-15T10:30:00Z
// Schema: 15 variables

declare namespace NodeJS {
  interface ProcessEnv {
    /** Application environment */
    NODE_ENV: 'development' | 'staging' | 'production';

    /** Application name */
    APP_NAME: string;

    /** Canonical application URL */
    APP_URL: string;

    /** Server listening port */
    PORT: number;

    /** Server bind address */
    HOST: string;

    /** Primary PostgreSQL connection string */
    DATABASE_URL: string;

    /** Database connection pool size */
    DATABASE_POOL_SIZE: number;

    /** JWT signing secret */
    JWT_SECRET: string;

    /** JWT token expiry duration */
    JWT_EXPIRY: string;

    /** Redis connection URL */
    REDIS_URL?: string;

    /** Enable analytics tracking */
    ENABLE_ANALYTICS: boolean;

    /** Application log level */
    LOG_LEVEL: 'debug' | 'info' | 'warn' | 'error' | 'fatal';

    /** Allowed CORS origins */
    ALLOWED_ORIGINS: readonly string[];

    /** Maximum file upload size */
    MAX_UPLOAD_SIZE: string;

    /** Optional admin email address */
    ADMIN_EMAIL?: string;
  }
}

export {};
```

### Usage

After generating the declaration file, you get full autocomplete:

```typescript
// process.env.PORT → number (not string!)
// process.env.NODE_ENV → 'development' | 'staging' | 'production'
// process.env.ADMIN_EMAIL → string | undefined

const port = process.env.PORT;          // Autocomplete! Type: number
const env = process.env.NODE_ENV;      // Autocomplete! Type: 'development' | 'staging' | 'production'
```

## TypeScript Modules

Generate a standalone TypeScript module with a typed `env` object:

### Generate

```bash
ultraenv typegen --format module --out src/env.ts
```

### Output

```typescript
// Auto-generated by ultraenv — DO NOT EDIT
// Generated: 2024-01-15T10:30:00Z
// Schema: 15 variables

export interface Env {
  /** Application environment */
  readonly NODE_ENV: 'development' | 'staging' | 'production';

  /** Application name */
  readonly APP_NAME: string;

  /** Server listening port */
  readonly PORT: number;

  // ... all variables
}

export const env: Env = {
  NODE_ENV: process.env.NODE_ENV as Env['NODE_ENV'],
  APP_NAME: process.env.APP_NAME as string,
  PORT: Number(process.env.PORT) as number,
  HOST: process.env.HOST as string,
  DATABASE_URL: process.env.DATABASE_URL as string,
  // ...
};

export default env;
```

### Usage

```typescript
import env from './env';

// All values are typed!
const port = env.PORT;          // number
const dbUrl = env.DATABASE_URL; // string
const debug = env.ENABLE_ANALYTICS; // boolean
```

## Watch Mode

Automatically regenerate types when your `.env` or schema changes:

```bash
ultraenv typegen --format declaration --out src/env.d.ts --watch
```

Watch mode:
1. Generates types on startup
2. Watches `.env`, `.env.local`, `.env.*` files for changes
3. Watches `.ultraenvrc.json` for config changes
4. Regenerates types when changes are detected

## JSDoc Generation

When `jsdoc: true` (default), ultraenv generates JSDoc comments from schema descriptions:

```typescript
// Schema definition
const env = defineEnv({
  PORT: t.number()
    .port()
    .description('Server listening port')
    .default(3000),

  DATABASE_URL: t.string()
    .format('url')
    .description('Primary PostgreSQL connection string')
    .required(),

  REDIS_URL: t.string()
    .format('url')
    .description('Redis connection URL (optional, enables caching)')
    .optional(),
});

// Generated .d.ts
declare namespace NodeJS {
  interface ProcessEnv {
    /** Server listening port */
    PORT: number;

    /** Primary PostgreSQL connection string */
    DATABASE_URL: string;

    /**
     * Redis connection URL (optional, enables caching)
     */
    REDIS_URL?: string;
  }
}
```

## Programmatic API

### generateDeclaration

```typescript
import { generateDeclaration } from 'ultraenv';

const content = generateDeclaration(envVars, schema, {
  interfaceName: 'Env',    // default: 'Env'
  jsdoc: true,             // default: true
  indent: 4,               // default: 4 (spaces)
  header: undefined,       // custom header comment
});
```

### generateModule

```typescript
import { generateModule } from 'ultraenv';

const content = generateModule(schema, {
  interfaceName: 'Env',    // default: 'Env'
  jsdoc: true,             // default: true
  indent: 2,               // default: 2 (spaces)
  header: undefined,       // custom header comment
  constAssertion: false,   // use `as const`
});
```

### generateJsonSchema

```typescript
import { generateJsonSchema } from 'ultraenv';

const schema = generateJsonSchema(schemaDefinition, {
  includeDescriptions: true,  // default: true
  indent: 2,                  // default: 2 (spaces)
});
```

### createTypegenWatcher

```typescript
import { createTypegenWatcher } from 'ultraenv';

const watcher = createTypegenWatcher({
  format: 'declaration',
  outFile: 'src/env.d.ts',
  schema: mySchema,
  watch: true,
});

watcher.start();
// Later: watcher.stop();
```

## Configuration

### CLI Options

| Option | Description | Default |
|---|---|---|
| `--format <fmt>` | `declaration`, `module`, or `json-schema` | `declaration` |
| `--out <path>` | Output file path | `src/env.d.ts` |
| `--interface <name>` | Interface name | `Env` |
| `--watch` | Watch for changes | `false` |

### Config File Options

```json
{
  "typegen": {
    "format": "declaration",
    "outFile": "src/env.d.ts",
    "interfaceName": "Env",
    "exportAsDefault": false,
    "constAssertion": false,
    "jsdoc": true,
    "indent": 4
  }
}
```

## Advanced Usage

### Custom Interface Name

```bash
ultraenv typegen --interface AppConfig --out src/config.d.ts
```

Generates `interface AppConfig` instead of `interface Env`.

### Multiple Schemas

Generate types from multiple schemas into different files:

```bash
ultraenv typegen --format declaration --out src/env.d.ts
ultraenv typegen --format module --out src/env.ts
ultraenv typegen --format json-schema --out env.schema.json
```

### tsconfig Integration

Ensure generated files are included in your TypeScript project:

```json
{
  "include": [
    "src/**/*.ts",
    "src/**/*.d.ts"
  ]
}
```

### With Next.js

For Next.js projects, generate types in the `src/` directory:

```bash
ultraenv typegen --format declaration --out src/env.d.ts
```

Then use in API routes:

```typescript
// pages/api/hello.ts
import type { NextApiRequest, NextApiResponse } from 'next';

export default function handler(req: NextApiRequest, res: NextApiResponse) {
  // process.env.DATABASE_URL is typed!
  const dbUrl = process.env.DATABASE_URL;
  res.status(200).json({ ok: true });
}
```
